# HR-PRIME — Парсер вакансий с hh.ru

Сервис для сбора, анализа и экспорта данных о вакансиях с hh.ru. Приложение собирает вакансии по регионам России, сохраняет данные в PostgreSQL и предоставляет REST API для дальнейшей работы с ними.

---

## 🚀 Возможности

* 🔍 Парсинг вакансий с hh.ru по регионам России;
* 💾 Сохранение вакансий и компаний в PostgreSQL;
* 🏢 Автоматическое создание компаний и подсчёт количества вакансий;
* 📊 Получение топа компаний по числу открытых вакансий;
* 📁 Экспорт данных в Excel;
* 🐳 Запуск приложения в Docker.

---

## 🏗 Архитектура приложения

Приложение построено по многослойной архитектуре:

```text
hh.ru
  ↓
Парсер (requests + BeautifulSoup)
  ↓
Нормализация данных (Pydantic-схемы)
  ↓
Repository (бизнес-логика)
  ↓
SQLAlchemy ORM
  ↓
PostgreSQL
  ↓
REST API / Excel Export
```

### Сбор данных

Парсер последовательно обходит указанные регионы России, получает HTML-страницы вакансий и извлекает следующую информацию:

* название вакансии;
* требуемый опыт работы;
* заработную плату;
* название компании;
* город.

### Нормализация данных

Полученные данные преобразуются в объекты `VacancySchema`, что позволяет привести все вакансии к единому формату перед сохранением.

### Сохранение данных

Слой `Repository` отвечает за:

* поиск существующих компаний;
* создание новых компаний;
* сохранение вакансий;
* подсчёт количества вакансий каждой компании.

Для уменьшения количества обращений к базе данных используется кэширование компаний во время обработки вакансий.

## ⚙️ Установка и запуск

### 1. Клонирование проекта

```bash
git clone https://github.com/Crack277/HR-PRIME-tz.git
cd HR-PRIME-tz
```

### 2. Создание виртуального окружения
```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install poetry

poetry install --no-root
```

### 4. Настройка окружения

Подразумевается, что пользователь запускает всё это дело через PgAdmin, поэтому в проекте имеется backup с данными.
Также можно запускать PostgreSQL через docker-compose, но данных в нём не будет.

Создайте файл `.env`:

```env
APP_CONFIG__DB__USERNAME=postgres
APP_CONFIG__DB__PASSWORD=postgres
APP_CONFIG__DB__HOST=localhost
APP_CONFIG__DB__PORT=5432
APP_CONFIG__DB__NAME=postgres
APP_CONFIG__DB__ECHO=false
```

## 🐳 Запуск PostgreSQL через Docker

## 📚 API Endpoints

| Метод | Endpoint                     | Описание                           |
| ----- | ---------------------------- | ---------------------------------- |
| POST  | `/v1/parser/vacancies`       | Сбор и сохранение вакансий с hh.ru |
| GET   | `/v1/parser/vacancies`       | Получение всех вакансий            |
| POST  | `/v1/parser/companies`       | Получение компаний с пагинацией    |
| POST  | `/v1/parser/vacancies_excel` | Экспорт данных в Excel             |

---

## 📁 Экспорт данных

При вызове эндпоинта:

```text
POST /v1/parser/vacancies_excel
```

создаётся файл:

```text
exports/companies.xlsx
```

Файл содержит два листа:

1. **Вакансии** — список компаний и их вакансий.
2. **Топ компаний** — компании, отсортированные по количеству вакансий.
